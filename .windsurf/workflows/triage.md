Move issues through a state machine of triage roles.

For the full skill with templates and reference docs, read `~/.agents/skills/triage/SKILL.md` and supporting files in that directory.

Every comment posted during triage must start with:
> *This was generated by AI during triage.*

## Roles

**Category:** `bug` | `enhancement`
**State:** `needs-triage` → `needs-info` | `ready-for-agent` | `ready-for-human` | `wontfix`

Every triaged issue carries exactly one category + one state role.

## Show What Needs Attention

Three buckets, oldest first:
1. **Unlabeled** — never triaged
2. **needs-triage** — evaluation in progress
3. **needs-info with reporter activity** — needs re-evaluation

## Triage a Specific Issue

1. **Gather context** — Read full issue, prior triage notes, explore codebase
2. **Recommend** — Category + state recommendation with reasoning
3. **Reproduce (bugs)** — Trace code, run tests, report what happened
4. **Grill (if needed)** — Run a `/grill-with-docs` session
5. **Apply outcome** — Post agent brief, triage notes, or close with explanation
